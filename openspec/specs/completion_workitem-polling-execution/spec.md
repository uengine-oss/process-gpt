# 워크아이템 폴링 실행 명세

## Purpose
시스템이 제출·대기 상태의 워크아이템을 주기적으로 감지하여 완료 여부를 판정하고, 다음 활동 워크아이템을 생성하며, 프로세스 인스턴스 상태를 전이시키는 능력을 보장한다.

## Requirements

### Requirement: 제출된 워크아이템 자동 폴링
시스템은 `SUBMITTED` 및 `PENDING` 상태의 워크아이템을 주기적으로 감지하여 처리 주체로 클레임하고 처리를 SHALL 시작한다.

#### Scenario: 제출 워크아이템 클레임
- **GIVEN** `status`가 `SUBMITTED`이고 처리 주체가 지정되지 않은 워크아이템이 존재한다
- **WHEN** 폴링 워커의 주기가 도래한다
- **THEN** 시스템은 해당 워크아이템을 처리 주체로 클레임하고 처리를 시작한다

### Requirement: 완료 판정과 다음 활동 진행
시스템은 처리한 워크아이템의 완료 여부를 판정하여 완료 활동을 `DONE` 또는 `PENDING`으로 전이하고, 다음 활동에 해당하는 워크아이템을 SHALL 생성한다.

#### Scenario: 워크아이템 완료 후 다음 활동 생성
- **GIVEN** 폴링 워커가 제출된 워크아이템을 처리한다
- **WHEN** 완료 조건이 충족된다
- **THEN** 시스템은 해당 활동을 `DONE`으로 전이하고 다음 활동 워크아이템을 `IN_PROGRESS` 등 적절한 상태로 생성한다

#### Scenario: 완료 조건 미충족
- **GIVEN** 폴링 워커가 워크아이템을 처리한다
- **WHEN** 체크포인트 등 진행 조건이 충족되지 않는다
- **THEN** 시스템은 해당 활동을 `PENDING`으로 전이하고 사유를 기록한다

#### Scenario: LLM 출력 파싱 실패
- **WHEN** 워크아이템 처리 중 모델 출력을 해석할 수 없다
- **THEN** 시스템은 워크아이템을 `PENDING`으로 전이하고 실패 사유를 시스템 메시지로 기록한다

### Requirement: 프로세스 인스턴스 상태 전이
시스템은 워크아이템 처리 결과에 따라 프로세스 인스턴스의 상태를 `RUNNING` 또는 `COMPLETED`로 SHALL 전이한다.

#### Scenario: 종료 활동 완료 시 인스턴스 완료
- **GIVEN** 인스턴스의 종료 활동 워크아이템이 완료된다
- **WHEN** 시스템이 인스턴스 상태를 갱신한다
- **THEN** 시스템은 인스턴스를 `COMPLETED`로 전이하고 종료 시각을 기록한다

#### Scenario: 진행 중 인스턴스 유지
- **GIVEN** 인스턴스에 아직 처리할 활동이 남아 있다
- **WHEN** 시스템이 인스턴스 상태를 갱신한다
- **THEN** 시스템은 인스턴스를 `RUNNING` 상태로 유지한다

### Requirement: 서브프로세스 생성과 부모 재개
시스템은 서브프로세스 활동을 처리할 때 자식 프로세스 인스턴스를 생성하고, 자식이 모두 완료되면 부모 워크아이템을 다시 진행 대상으로 SHALL 만든다.

#### Scenario: 서브프로세스 자식 인스턴스 생성
- **WHEN** 폴링 워커가 서브프로세스 활동을 처리한다
- **THEN** 시스템은 자식 프로세스 인스턴스와 초기 워크아이템을 생성한다

#### Scenario: 자식 완료 시 부모 재개
- **GIVEN** 한 부모의 모든 자식 인스턴스가 완료되었다
- **WHEN** 시스템이 대기 중인 부모 워크아이템을 평가한다
- **THEN** 시스템은 부모 워크아이템을 다시 진행 대상으로 전이한다

### Requirement: 처리 실패 재시도 제한
시스템은 워크아이템 처리 실패 시 재시도 횟수를 증가시키고, 재시도 한도를 초과하면 워크아이템을 강제로 종료 처리 SHALL 한다.

#### Scenario: 실패 후 재시도
- **WHEN** 워크아이템 처리가 실패한다
- **THEN** 시스템은 재시도 횟수를 증가시키고 처리 주체 클레임을 해제하여 재처리 대상으로 되돌린다

#### Scenario: 재시도 한도 초과
- **GIVEN** 워크아이템의 재시도 횟수가 한도(3회)를 초과한다
- **WHEN** 처리가 다시 실패한다
- **THEN** 시스템은 워크아이템을 `DONE`으로 강제 종료하고 오류 로그를 기록한다

### Requirement: 외부 고객 대상 이메일 안내
시스템은 다음 활동의 담당자가 외부 고객(`external_customer`)인 경우 해당 고객에게 작업 진행 안내 이메일을 SHALL 발송한다.

#### Scenario: 외부 고객 안내 이메일 발송
- **GIVEN** 다음 활동의 담당 역할 endpoint가 `external_customer`이고 완료된 데이터에서 고객 이메일을 확인할 수 있다
- **WHEN** 시스템이 다음 활동을 진행시킨다
- **THEN** 시스템은 외부 폼 링크가 포함된 안내 이메일을 고객에게 발송한다

### Requirement: 정체된 처리 클레임 회수
시스템은 처리 주체가 클레임한 채 일정 시간 진행되지 않은 워크아이템의 클레임을 SHALL 회수한다.

#### Scenario: 정체 워크아이템 회수
- **GIVEN** `SUBMITTED` 워크아이템이 처리 주체에 클레임된 채 30분 넘게 갱신되지 않았다
- **WHEN** 정리 주기가 도래한다
- **THEN** 시스템은 해당 워크아이템의 클레임을 해제하여 다시 처리 대상으로 만든다
