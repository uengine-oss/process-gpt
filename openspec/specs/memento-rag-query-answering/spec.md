# RAG 질의응답 명세

## Purpose
클라이언트가 자연어 질문을 보내면 관련 문서를 검색해 LLM으로 답변을 생성하고, 답변과 출처 메타데이터를 함께 반환하는 능력을 보장한다.

## Requirements

### Requirement: 검색 기반 질의 답변
시스템은 `GET /query` 요청을 받으면 테넌트 범위의 문서를 검색해 그 컨텍스트로 답변을 생성하고, 답변(`response`)과 출처 메타데이터(`metadata`)를 SHALL 반환한다.

#### Scenario: 관련 문서가 있는 질문
- **GIVEN** 테넌트에 질문과 관련된 문서가 적재되어 있다
- **WHEN** 클라이언트가 `query`, `tenant_id`로 `GET /query`를 호출한다
- **THEN** 시스템은 검색된 컨텍스트를 근거로 생성한 답변을 `response`로 반환한다
- **AND** `metadata`에는 `파일명#청크인덱스` 키별 출처 문서 메타데이터가 포함된다

#### Scenario: 관련 문서가 없는 질문
- **GIVEN** 질문과 관련된 문서가 테넌트에 없다
- **WHEN** 클라이언트가 `GET /query`를 호출한다
- **THEN** 시스템은 답을 지어내지 않고 정보가 충분하지 않다는 안내를 답변으로 반환한다
- **AND** 출처 문서 목록은 비어 있다

#### Scenario: 처리 중 오류
- **GIVEN** 답변 생성 과정에서 오류가 발생한다
- **WHEN** 클라이언트가 `GET /query`를 호출한다
- **THEN** 시스템은 `500` 상태의 오류 또는 질문 처리 중 오류가 발생했다는 안내 답변을 반환한다

### Requirement: 질문 언어에 맞춘 답변
시스템은 질문에 한글이 포함되어 있으면 한국어로, 그렇지 않으면 영어로 답변과 안내 문구를 SHALL 생성한다.

#### Scenario: 한국어 질문
- **GIVEN** 클라이언트가 한글이 포함된 질문을 보낸다
- **WHEN** 클라이언트가 `GET /query`를 호출한다
- **THEN** 시스템은 한국어로 답변하며, 정보가 없을 때의 안내도 한국어로 제공한다

#### Scenario: 영어 질문
- **GIVEN** 클라이언트가 한글이 없는 질문을 보낸다
- **WHEN** 클라이언트가 `GET /query`를 호출한다
- **THEN** 시스템은 영어로 답변하며, 정보가 없을 때의 안내도 영어로 제공한다

### Requirement: 테넌트 격리 답변
시스템은 질의 답변의 컨텍스트로 요청한 `tenant_id`에 속한 문서만 SHALL 사용한다.

#### Scenario: 다른 테넌트 문서 비노출
- **GIVEN** 동일 질의와 관련된 문서가 다른 테넌트에만 존재한다
- **WHEN** 클라이언트가 자신의 `tenant_id`로 `GET /query`를 호출한다
- **THEN** 시스템은 다른 테넌트의 문서를 컨텍스트나 출처로 사용하지 않는다
