# 인스턴스 인제스천 명세

## Purpose
`instance-classifier` 서비스가 새로 생성된 프로세스 인스턴스를 주기적으로 감지하여, 첫(시작) 워크아이템 폼에 입력된 **사용자 요청 내용**만 뽑아 분석용 저장소에 적재하는 능력을 보장한다. 폼의 골격(양식명·확인 체크리스트·이메일·날짜 등 공통 필드)이 아니라 실제 요청 내용에 집중한다.

## Requirements

### Requirement: 상태 점검
시스템은 `GET /health` 요청에 대해 서비스 동작 상태와 설정 요약을 SHALL 반환한다.

#### Scenario: 헬스 응답
- **WHEN** 클라이언트가 `GET /health`를 호출한다
- **THEN** 시스템은 `status`, `db`, `poller`, `config` 필드를 포함한 JSON을 반환한다

### Requirement: 신규 인스턴스 폴링 적재
시스템은 프로세스 인스턴스 원천(`bpm_proc_inst`)에서 아직 인제스천하지 않은 인스턴스를 주기적으로 감지하여 인스턴스당 한 건씩 적재하고, 각 인스턴스의 생성 시각을 함께 저장 SHALL 한다.

#### Scenario: 새 인스턴스 자동 인제스천
- **GIVEN** 원천에 아직 인제스천되지 않은 프로세스 인스턴스가 존재한다
- **WHEN** 폴링 주기가 도래한다
- **THEN** 시스템은 해당 인스턴스의 첫 워크아이템에서 요청 텍스트를 추출해 적재하고, 인스턴스 생성 시각을 `occurred_at`으로 저장한다

#### Scenario: 이미 적재된 인스턴스 재적재 방지
- **GIVEN** 어떤 인스턴스가 이미 적재되어 있다
- **WHEN** 다음 폴링이 실행된다
- **THEN** 시스템은 그 인스턴스를 다시 적재하지 않는다(멱등)

#### Scenario: 요청 내용이 아직 없는 인스턴스
- **GIVEN** 첫 워크아이템 폼에 아직 입력 내용이 없다
- **WHEN** 폴링이 그 인스턴스를 검사한다
- **THEN** 시스템은 적재를 건너뛰고 다음 폴링에서 다시 시도한다

### Requirement: 내용 중심 텍스트 추출
시스템은 첫 워크아이템 폼 값에서 **자유텍스트 내용 필드만** 사용하고, 이메일·날짜·체크박스·선택·파일 등 비내용 필드와 체크리스트(배열) 값, 필드 라벨 접두어는 요청 텍스트에서 SHALL 제외한다. 내용을 뜻하는 필드(예: 민원내용/content)가 있으면 그 필드를 우선 사용한다.

#### Scenario: 폼 골격 제외
- **GIVEN** 폼 출력이 확인 체크리스트, `customer_email`, `complaint_date`, `complaint_content`를 포함한다
- **WHEN** 시스템이 요청 텍스트를 구성한다
- **THEN** 요청 텍스트는 `complaint_content`의 값만 담고 체크리스트·이메일·날짜·라벨은 포함하지 않는다

#### Scenario: 폼 정의가 없을 때 폴백
- **GIVEN** 폼 필드 정의를 조회할 수 없다
- **WHEN** 시스템이 요청 텍스트를 구성한다
- **THEN** 시스템은 이메일 형태·날짜 형태·배열 값을 제외한 자유텍스트 값만 사용한다

### Requirement: 단건 인제스천
시스템은 `POST /ingest`로 특정 인스턴스를 즉시 인제스천 SHALL 한다.

#### Scenario: 수동 단건 인제스천
- **WHEN** 클라이언트가 `POST /ingest`에 `proc_inst_id`(선택 `force`)를 담아 호출한다
- **THEN** 시스템은 해당 인스턴스를 인제스천하고 배정된 유형 정보를 반환하거나, 요청 내용이 없으면 `skipped`를 반환한다

### Requirement: 재처리
시스템은 `POST /reingest`로 한 프로세스 정의의 모든 인스턴스를 원천에서 다시 읽어 요청 텍스트와 임베딩을 갱신 SHALL 한다.

#### Scenario: 추출/임베딩 변경 후 재처리
- **WHEN** 클라이언트가 `POST /reingest`에 `proc_def_id`를 담아 호출한다
- **THEN** 시스템은 그 정의의 모든 인스턴스를 강제로 다시 인제스천하고 처리·건너뜀 건수를 반환한다

### Requirement: 적재된 정의 목록
시스템은 `GET /proc-defs` 요청에 대해 인제스천된 프로세스 정의 목록과 각 건수·유형 수를 SHALL 반환한다.

#### Scenario: 정의 목록 조회
- **WHEN** 클라이언트가 `GET /proc-defs`를 호출한다
- **THEN** 시스템은 `proc_def_id`, `count`, `topics`를 담은 목록을 반환한다
