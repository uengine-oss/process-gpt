# CrewAI 액션 리포트·슬라이드 전달 명세

## Purpose
작업의 폼 정의에 리포트(문서) 또는 슬라이드(프레젠테이션) 타입 필드가 포함될 때, 해당 산출물을 일반 폼 데이터와 분리하여 전용 이벤트로 전달하도록 보장한다.

## Requirements

### Requirement: 리포트 타입 필드의 분리 이벤트 전달
시스템은 작업의 폼 정의에 타입이 `report` 또는 `document`인 필드가 있고 그 값이 비어 있지 않으면, 해당 필드를 전용 리포트 이벤트로 SHALL 전달한다. 각 리포트 필드는 메타데이터 `crew_type`이 `report`인 `working`(`event_type: task_started`)과 `completed`(`event_type: task_completed`) 상태 이벤트 쌍으로 발행되며, 완료 이벤트 본문은 해당 폼 필드의 `key`를 키로 사용한다.

#### Scenario: 리포트 필드 결과 전달
- **GIVEN** 폼 정의에 타입이 `report`인 필드가 있고 실행 결과에 그 필드의 값이 채워졌다
- **WHEN** 시스템이 결과를 전달한다
- **THEN** 시스템은 `crew_type`이 `report`인 `working`과 `completed` 이벤트 쌍을 발행한다
- **AND** 완료 이벤트 본문은 `{필드 key: 마크다운 값}` 형식이다

#### Scenario: 값이 비어 있는 리포트 필드
- **GIVEN** 폼 정의에 타입이 `report`인 필드가 있으나 실행 결과에 그 필드의 값이 비어 있다
- **WHEN** 시스템이 결과를 전달한다
- **THEN** 시스템은 해당 리포트 필드에 대한 이벤트를 발행하지 않는다

### Requirement: 슬라이드 타입 필드의 분리 이벤트 전달
시스템은 작업의 폼 정의에 타입이 `slide` 또는 `presentation`인 필드가 있고 그 값이 비어 있지 않으면, 해당 필드를 전용 슬라이드 이벤트로 SHALL 전달한다. 각 슬라이드 필드는 메타데이터 `crew_type`이 `slide`인 `working`(`event_type: task_started`)과 `completed`(`event_type: task_completed`) 상태 이벤트 쌍으로 발행되며, 완료 이벤트 본문은 해당 폼 필드의 `key`를 키로 사용한다.

#### Scenario: 슬라이드 필드 결과 전달
- **GIVEN** 폼 정의에 타입이 `slide`인 필드가 있고 실행 결과에 그 필드의 값이 채워졌다
- **WHEN** 시스템이 결과를 전달한다
- **THEN** 시스템은 `crew_type`이 `slide`인 `working`과 `completed` 이벤트 쌍을 발행한다
- **AND** 완료 이벤트 본문은 `{필드 key: 마크다운 값}` 형식이다

#### Scenario: 값이 비어 있는 슬라이드 필드
- **GIVEN** 폼 정의에 타입이 `slide`인 필드가 있으나 실행 결과에 그 필드의 값이 비어 있다
- **WHEN** 시스템이 결과를 전달한다
- **THEN** 시스템은 해당 슬라이드 필드에 대한 이벤트를 발행하지 않는다

### Requirement: 리포트·슬라이드 필드의 일반 폼 데이터 제외
시스템은 리포트·슬라이드 타입 필드를 일반 폼 결과 이벤트의 `폼_데이터`에서 SHALL 제외한다. 해당 필드는 전용 리포트·슬라이드 이벤트로만 전달되며, 일반 폼 결과 데이터에 중복 포함되지 않는다.

#### Scenario: 일반 폼 결과에서 리포트·슬라이드 필드 분리
- **GIVEN** 실행 결과에 리포트 타입 필드와 일반 폼 필드가 함께 포함되어 있다
- **WHEN** 시스템이 결과를 전달한다
- **THEN** 일반 폼 결과 이벤트의 `폼_데이터`에는 리포트 타입 필드가 포함되지 않는다
- **AND** 리포트 타입 필드는 전용 리포트 이벤트로만 전달된다
