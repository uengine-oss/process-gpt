# 자동 작업 실행 명세

## Purpose
시스템이 사람이 개입하지 않는 프로세스 활동을 자동으로 실행하는 능력을 보장한다. 서비스 작업은 테넌트별 MCP 도구로, 스크립트 작업은 파이썬 코드로 실행하고 그 결과를 워크아이템에 기록한다.

## Requirements

### Requirement: 서비스 작업의 MCP 도구 실행
시스템은 서비스 작업(serviceTask) 워크아이템을 처리할 때 테넌트별 MCP 설정에 따라 지정된 도구를 실행하고 그 결과를 워크아이템에 SHALL 기록한다.

#### Scenario: 서비스 작업 도구 실행 성공
- **GIVEN** `serviceTask` 유형의 `SUBMITTED` 워크아이템이 있고 해당 활동에 사용할 도구가 지정되어 있다
- **WHEN** 폴링 워커가 워크아이템을 처리한다
- **THEN** 시스템은 테넌트 MCP 설정에 매칭되는 도구를 실행하고 도구별 `status`와 연결 정보를 담은 결과를 워크아이템 `output`에 기록한다
- **AND** 워크아이템 상태를 `DONE`으로 전이하고 다음 활동을 진행시킨다

#### Scenario: 도구 실행 실패 기록
- **WHEN** MCP 도구 실행 중 일부 또는 전체 도구가 실패한다
- **THEN** 시스템은 실패한 도구의 오류 정보를 결과에 포함하고 처리 로그에 실패 내용을 기록한다

### Requirement: 스크립트 작업의 코드 실행
시스템은 스크립트 작업(scriptTask) 워크아이템을 처리할 때 활동에 정의된 파이썬 코드를 실행하고 실행 결과를 워크아이템에 SHALL 기록한다.

#### Scenario: 스크립트 작업 코드 실행 성공
- **GIVEN** `scriptTask` 활동에 실행할 파이썬 코드가 정의되어 있다
- **WHEN** 폴링 워커가 해당 작업을 실행한다
- **THEN** 코드가 정상 종료되면 시스템은 표준 출력을 결과로 기록하고 활동을 `DONE`으로 전이한다

#### Scenario: 스크립트 작업 코드 실행 실패
- **GIVEN** `scriptTask` 코드가 비정상 종료된다
- **WHEN** 폴링 워커가 해당 작업을 실행한다
- **THEN** 시스템은 오류 출력을 결과로 기록하고 오류 경로에 따라 다음 활동을 진행시킨다
