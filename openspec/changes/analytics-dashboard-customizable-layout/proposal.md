## Why

분석 대시보드는 현재 모든 사용자에게 고정된 위젯 구성과 배치로만 제공된다. 통계 카드, 작업 분포, 월별 추이, FTE 히트맵, 부서별 작업량, Agent-대-Human 비교, 프로세스 성능, 병목 분석, 최근 인스턴스 표 등 위젯이 항상 같은 순서·같은 위치로 나타나므로, 역할이나 관심 지표가 다른 사용자가 원하는 지표를 앞에 두거나 불필요한 위젯을 숨길 수 없다. 사용자가 대시보드를 자신의 업무에 맞게 재구성하고 그 구성을 유지할 수 있어야 한다.

## What Changes

- 분석 대시보드의 위젯을 **드래그로 위치 이동**하고 순서를 바꿀 수 있게 한다.
- 개별 위젯을 **숨기거나(제거) 다시 추가**할 수 있는 위젯 팔레트/구성 메뉴를 제공한다.
- 위젯의 **크기(열 너비/행 높이 등 그리드 단위)를 조정**할 수 있게 한다.
- 사용자가 만든 레이아웃(위젯 목록·순서·위치·크기·표시 여부)을 **사용자별로 저장하고, 다음 접속 시 복원**한다.
- 레이아웃을 **기본값으로 초기화**하는 기능을 제공한다.
- 저장된 레이아웃이 없거나 손상된 경우 **기본 레이아웃으로 안전하게 폴백**한다.
- 위젯 구성에 새 위젯 종류가 추가/폐기되어도 저장된 레이아웃이 **하위 호환**되도록 한다(알 수 없는 위젯은 무시, 신규 위젯은 기본 위치로 노출).

## Capabilities

### New Capabilities
- `analytic-service_dashboard-layout`: 분석 대시보드 위젯의 배치·크기·표시 여부를 사용자별로 편집하고, 저장·복원·초기화하는 개인화 레이아웃 기능. 편집 모드 진입/종료, 드래그 이동, 위젯 추가/제거, 레이아웃 영속화(저장/조회), 기본값 폴백을 다룬다.

## Impact

- **화면**: 분석 대시보드 화면(`/analytics`) — 편집 모드 토글, 드래그 그리드, 위젯 구성 메뉴, 초기화 버튼이 추가된다. 각 위젯은 이동/숨김/리사이즈가 가능한 그리드 항목으로 감싸진다.
- **API/영속성**: 사용자별 대시보드 레이아웃을 저장/조회하는 지속성 계약이 추가된다(레이아웃 저장, 레이아웃 조회, 기본값 초기화). 저장 위치와 API 노출 방식(analytic-service 엔드포인트 또는 Supabase 직접 접근)은 design 단계에서 확정한다.
- **데이터**: 사용자별 레이아웃 구성을 담는 저장소(사용자 식별자·테넌트·위젯 배치 문서·버전)가 필요하다. 위젯이 표시하는 지표 데이터 소스 자체(분석용 뷰·테이블)는 이 변경의 범위가 아니다.
- **범위 밖**: 각 위젯이 보여주는 지표의 계산 로직, 분석용 뷰/테이블 스키마, 위젯 데이터 조회 엔드포인트는 변경하지 않는다. 이 변경은 "어떤 위젯을 어디에 어떻게 배치하는가"의 개인화만 다룬다.
