## ADDED Requirements

### Requirement: 대시보드 편집 모드 진입 및 종료

분석 대시보드는 사용자가 레이아웃을 편집할 수 있는 편집 모드를 제공해야 한다(SHALL). 기본 상태는 조회 모드이며, 편집 모드에서만 위젯 이동·크기 조정·추가·제거 조작이 활성화된다. 편집 모드가 아닐 때는 위젯이 일반 조회 상태로 표시되어 데이터 조회와 상호작용에 영향을 주지 않아야 한다(SHALL).

#### Scenario: 편집 모드로 전환
- **WHEN** 사용자가 대시보드에서 편집 모드 토글을 활성화하면
- **THEN** 각 위젯에 이동 핸들, 크기 조정 핸들, 제거 컨트롤이 나타나고, 위젯 추가를 위한 구성 메뉴에 접근할 수 있다

#### Scenario: 편집 모드 종료
- **WHEN** 사용자가 편집 모드를 종료하면
- **THEN** 이동·크기·제거 컨트롤이 사라지고 대시보드는 조회 모드로 돌아가며, 현재까지 반영된 배치가 그대로 유지된다

#### Scenario: 조회 모드에서는 배치 변경 불가
- **WHEN** 대시보드가 조회 모드일 때 사용자가 위젯 위에서 드래그를 시도하면
- **THEN** 위젯 위치와 크기는 변경되지 않는다

### Requirement: 위젯 위치 이동 및 순서 변경

편집 모드에서 사용자는 위젯을 드래그하여 대시보드 그리드 내 다른 위치로 이동하고 표시 순서를 바꿀 수 있어야 한다(SHALL). 이동 중에는 대상 위치를 시각적으로 안내하고, 다른 위젯과 겹치지 않도록 배치를 재정렬해야 한다(SHALL).

#### Scenario: 위젯을 다른 위치로 이동
- **WHEN** 사용자가 편집 모드에서 한 위젯을 드래그하여 다른 위치에 놓으면
- **THEN** 해당 위젯이 놓인 위치로 이동하고, 영향을 받는 다른 위젯들이 겹치지 않도록 재배치된다

#### Scenario: 이동 중 배치 미리보기
- **WHEN** 사용자가 위젯을 드래그하는 동안
- **THEN** 위젯이 놓일 대상 위치가 시각적으로 표시된다

### Requirement: 위젯 크기 조정

편집 모드에서 사용자는 위젯의 크기(그리드 열·행 단위)를 조정할 수 있어야 한다(SHALL). 각 위젯에는 정의된 최소 크기가 있으며, 그 이하로는 축소되지 않아야 한다(SHALL). 크기 변경 시 그리드 열 수를 초과하지 않도록 제한해야 한다(SHALL).

#### Scenario: 위젯 크기 확대
- **WHEN** 사용자가 편집 모드에서 위젯의 크기 조정 핸들을 드래그하여 크기를 늘리면
- **THEN** 위젯이 새 크기로 확장되고 주변 위젯이 겹치지 않도록 재배치된다

#### Scenario: 최소 크기 이하 축소 방지
- **WHEN** 사용자가 위젯을 정의된 최소 크기보다 작게 축소하려 하면
- **THEN** 위젯은 최소 크기까지만 축소되고 더 작아지지 않는다

### Requirement: 위젯 추가 및 제거

편집 모드에서 사용자는 개별 위젯을 대시보드에서 제거하여 숨기고, 구성 메뉴(위젯 팔레트)에서 숨겨진 위젯을 다시 추가할 수 있어야 한다(SHALL). 구성 메뉴는 현재 표시 중인 위젯과 숨겨진 위젯을 구분하여 보여줘야 한다(SHALL).

#### Scenario: 위젯 제거
- **WHEN** 사용자가 편집 모드에서 특정 위젯의 제거 컨트롤을 실행하면
- **THEN** 해당 위젯이 대시보드에서 사라지고, 구성 메뉴에서 숨겨진 위젯 목록에 나타난다

#### Scenario: 숨긴 위젯 다시 추가
- **WHEN** 사용자가 구성 메뉴에서 숨겨진 위젯을 선택하여 추가하면
- **THEN** 해당 위젯이 대시보드에 다시 표시되고 다른 위젯과 겹치지 않는 위치에 배치된다

### Requirement: 사용자별 레이아웃 저장 및 복원

시스템은 사용자가 구성한 레이아웃(표시 위젯 목록, 각 위젯의 위치·크기·표시 여부)을 사용자별로 저장해야 한다(SHALL). 사용자가 대시보드에 다시 접속하면 저장된 레이아웃을 복원하여 표시해야 한다(SHALL). 레이아웃은 사용자 식별자와 테넌트 기준으로 격리되어, 한 사용자의 레이아웃이 다른 사용자에게 노출되지 않아야 한다(SHALL).

#### Scenario: 레이아웃 저장
- **WHEN** 사용자가 편집한 레이아웃을 저장하면
- **THEN** 해당 사용자에 대해 위젯 배치 구성이 영속적으로 보관되고, 저장 성공이 사용자에게 확인된다

#### Scenario: 재접속 시 레이아웃 복원
- **WHEN** 저장된 레이아웃이 있는 사용자가 분석 대시보드를 다시 열면
- **THEN** 마지막으로 저장한 위젯 배치·크기·표시 여부가 그대로 복원되어 표시된다

#### Scenario: 사용자 간 레이아웃 격리
- **WHEN** 서로 다른 두 사용자가 각각 레이아웃을 저장한 뒤 대시보드를 열면
- **THEN** 각 사용자는 자신이 저장한 레이아웃만 보고, 다른 사용자의 레이아웃은 보이지 않는다

### Requirement: 기본 레이아웃 초기화

사용자는 자신의 대시보드 레이아웃을 시스템 기본 레이아웃으로 초기화할 수 있어야 한다(SHALL). 초기화하면 사용자가 저장했던 개인 레이아웃은 제거되고, 기본 위젯 구성으로 되돌아가야 한다(SHALL).

#### Scenario: 레이아웃 초기화
- **WHEN** 사용자가 레이아웃 초기화를 실행하면
- **THEN** 모든 위젯이 기본 순서·위치·크기·표시 상태로 복원되고, 이후 재접속 시에도 기본 레이아웃이 적용된다

### Requirement: 저장된 레이아웃의 안전한 폴백과 하위 호환

시스템은 저장된 레이아웃이 없거나, 손상되었거나, 현재 위젯 구성과 일치하지 않는 경우에도 대시보드를 안전하게 표시해야 한다(SHALL). 저장된 레이아웃이 없으면 기본 레이아웃을 사용하고, 저장된 레이아웃에 더 이상 존재하지 않는 위젯이 포함되어 있으면 해당 항목을 무시하며, 저장된 레이아웃에 포함되지 않은 신규 위젯은 기본 위치로 노출해야 한다(SHALL).

#### Scenario: 저장된 레이아웃 없음
- **WHEN** 저장된 개인 레이아웃이 없는 사용자가 대시보드를 열면
- **THEN** 시스템 기본 레이아웃으로 모든 기본 위젯이 표시된다

#### Scenario: 손상되었거나 읽을 수 없는 레이아웃
- **WHEN** 저장된 레이아웃을 해석할 수 없는 경우
- **THEN** 오류로 화면이 비어 있는 대신 기본 레이아웃으로 폴백하여 대시보드가 정상 표시된다

#### Scenario: 알 수 없는 위젯 무시 및 신규 위젯 노출
- **WHEN** 저장된 레이아웃에 더 이상 제공되지 않는 위젯이 포함되어 있고, 새로 추가된 위젯은 저장된 레이아웃에 없을 때
- **THEN** 더 이상 제공되지 않는 위젯은 무시되고, 새 위젯은 기본 위치에 표시되며, 나머지 저장된 배치는 유지된다
